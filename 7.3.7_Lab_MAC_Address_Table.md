# 7.3.7 Lab – View the Switch MAC Address Table


Lab focus: Building a simple Layer 2 network and observing how switches dynamically learn and store MAC addresses.



## Objectives

### Part 1: Build and Configure the Network
- Set up a topology with two switches and two PCs
- Configure IP addressing on all devices
- Establish connectivity between hosts

### Part 2: Examine the Switch MAC Address Table
- Record MAC addresses of connected devices
- Observe how switches learn MAC addresses
- Analyze MAC address table behavior before and after network communication


## Network Topology

- PC-A connected to Switch S1
- S1 connected to S2
- S2 connected to PC-B

## Configuration Steps

### 1. Configure PC IP Addresses

**PC-A**
IP Address: 192.168.1.1  
Subnet Mask: 255.255.255.0  

**PC-B**

Subnet Mask: 255.255.255.0

## Key Concepts

### MAC Address
A unique hardware identifier assigned to a network interface.

### MAC Address Table
A table maintained by the switch that maps MAC addresses to switch ports.

### ARP (Address Resolution Protocol)
Resolves IP addresses to MAC addresses.

### Switch Learning Process
- Learns MAC addresses from incoming frames (source MAC)
- Stores MAC-to-port mappings
- Uses this information to forward frames efficiently

---

## Observations

- Initially, the MAC address table may be empty or partially populated
- After generating traffic (ping), the switch learns and stores MAC addresses
- Clearing the MAC table removes entries, but they are relearned when traffic resumes

## Results

- Successful communication established between PC-A and PC-B
- Switches dynamically learned MAC addresses
- MAC address table updated correctly after network activity

## Challenges

- Identifying devices based solely on MAC addresses
- Managing large MAC address tables in bigger networks
- Increased broadcast traffic when MAC addresses are unknown
- Security concerns such as MAC spoofing
